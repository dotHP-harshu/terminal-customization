What this does

Two layers of customization

This project replaces the default PowerShell experience with a terminal that shows system information on startup and presents a styled, informative prompt every time you press Enter.

Layer 1

ASCII Startup Banner

When you open a terminal, Fastfetch displays your custom ASCII art. The logo uses Catppuccin Mocha colors and renders with JetBrains Mono Nerd Font.

Before you start

Prerequisites

Three things you need installed before proceeding. The font is mandatory, the rest are recommended.

Required

JetBrains Mono Nerd Font

Without this, icons render as empty boxes. Download from nerdfonts.com and install the .ttf files.

Recommended

Fastfetch

Displays system info with your ASCII logo on startup. Install via winget.

winget install fastfetch
Required

PowerShell 7+

The prompt scripts use features only available in PowerShell 7 or later. Check with $PSVersionTable.

Step by step

Setup guide

Follow these four steps in order. Each one builds on the previous.

Install the Nerd Font

Go to nerdfonts.com/font-downloads, search for JetBrainsMono, download the zip, extract it, and install every .ttf file. Then set your terminal to use "JetBrainsMono Nerd Font" in its settings.

Install Fastfetch and configure it

Fastfetch reads your ASCII art and system info on terminal startup. Install it, then create the config directory and copy the two files from this repo.

PowerShell 
winget install fastfetch
PowerShell 
mkdir "$HOME\.config\fastfetch"
PowerShell 

# Copy from this repo:
#   fastfetch/ascii.txt      -> your ASCII logo
#   fastfetch/config.jsonc   -> display config

You can find ./fastfetch/ in github.
The config file points to ~/.config/fastfetch/ascii.txt and applies Catppuccin Mocha colors. Edit ascii.txt with your own art if desired.

Set up your PowerShell profile

Your PowerShell profile runs every time you open a terminal. This is where the startup banner and custom prompt live.

PowerShell 
# Find your profile path
$PROFILE

# Create it if it doesn't exist
if (!(Test-Path -Path $PROFILE)) {
    New-Item -ItemType File -Path $PROFILE -Force
}

# Open in Notepad
notepad $PROFILE

Delete everything in that file, then paste the contents of your chosen prompt file from the standalone/ folder. Save and close.

Restart your terminal

Close and reopen Windows Terminal. You should see the Fastfetch banner followed by your new styled prompt. To test the clear shortcut, type cls or clear and press Enter.

Make it yours

Customization

Every prompt file defines its color palette at the top. Here is what each variable controls and how to change it.

Color Variables

Each standalone file begins with a block of $C_* variables. These map to PowerShell's [ConsoleColor] enum.

$C_SUCCESS Prompt character, clean indicators
$C_DIR Folder and directory icons
$C_GIT Git branch icon
$C_BRANCH Git branch name
$C_USER Username
$C_DIRTY Dirty git indicator (*)

Available colors: White, Black, DarkGray, Gray, Red, DarkRed, Green, DarkGreen, Blue, DarkBlue, Cyan, DarkCyan, Magenta, DarkMagenta, Yellow, DarkYellow.

Path Color Cycle

The $C_PATH_CYCLE array controls the colors for each folder level in the path. Each segment gets the next color in the list.

Default cycle 
$C_PATH_CYCLE = @(
    [ConsoleColor]::Blue,
    [ConsoleColor]::Cyan,
    [ConsoleColor]::Magenta,
    [ConsoleColor]::Yellow
)

Add, remove, or reorder colors. The cycle repeats if the path has more segments than colors.

Username and Hostname

Most prompts read from environment variables automatically. To hardcode a different name, replace $env:USERNAME with a string.

Custom name 
# Default (reads from system)
Write-Host $env:USERNAME ...

# Custom name
Write-Host "harsh" ...

Under the hood

How it works

The startup flow is straightforward. Here is what happens from the moment you open a terminal.

Terminal Opens
Windows Terminal launches PowerShell 7
$PROFILE Loads
PowerShell reads your profile script from Documents\PowerShell\
Clear-Host + Show-Fastfetch
Screen clears, Fastfetch runs with your config and ASCII art
prompt Function Renders
Custom prompt draws username, path, git, and other info
Ready for Input
You type commands. On cls/clear, Show-Fastfetch re-runs.

VS Code Integration

To make VS Code's integrated terminal also show Fastfetch on startup, add a custom profile to your VS Code settings.json.

settings.json 
{
  "terminal.integrated.profiles.windows": {
    "PowerShell": {
      "source": "PowerShell",
      "args": [
        "-NoExit",
        "-Command",
        "fastfetch -c 'C:/Users/YourName/.config/fastfetch/config.jsonc'"
      ]
    }
  }
}

Replace C:/Users/YourName/ with your actual home path. Run $HOME in PowerShell to find it.

Troubleshooting

Frequently asked questions

You have not installed the Nerd Font, or your terminal is not set to use it. Install JetBrains Mono Nerd Font from nerdfonts.com, then go to your terminal settings and set the font to "JetBrainsMono Nerd Font". Restart the terminal after changing.

Check if it is installed by running fastfetch --version in PowerShell. If not found, install it with winget install fastfetch. Also verify that your profile includes the Show-Fastfetch function and that it is called at the top of the profile script.

Open $PROFILE in Notepad and check for syntax errors. PowerShell is strict about string escaping and brackets. If you are stuck, delete everything and paste a fresh copy of your chosen standalone file.

Make sure your profile includes the alias lines at the bottom:

function c {
    Clear-Host
    Show-Fastfetch
}

Set-Alias cls c
Set-Alias clear c

The $C_PATH_CYCLE array might be empty or contain invalid color names. Make sure each entry is a valid [ConsoleColor] value. The cycle repeats if your path has more segments than the number of colors defined.

The prompt scripts work in any PowerShell 7+ terminal. The Fastfetch startup banner works in any terminal that supports it. Windows Terminal is recommended for the best experience with themes and transparency, but ConEmu, Cmder, and VS Code's integrated terminal all work.
